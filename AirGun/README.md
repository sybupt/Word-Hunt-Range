## ✨ 特性

- **高精度体感瞄准**：基于 MPU6050 六轴陀螺仪，支持可配置增益、死区、平滑滤波和 Y 轴反转。
- **动态零漂校准**：静止时自动更新陀螺仪偏置，消除温漂和长时间累积误差。
- **双核任务架构**：传感器读取与 BLE 发送分离，保证 200Hz 采样率的同时避免蓝牙卡顿影响采集。
- **防累积限幅**：pending 位移超过上限时自动截断，防止 BLE 严重阻塞后鼠标瞬移。
- **低功耗设计**：未连接时 BLE 任务挂起，连接后按固定节拍发送，减少空转。
- **电池电量上报**：通过 ADC 读取电池电压，自动转换为百分比并通过蓝牙上报主机。
- **按键消抖**：使用 Bounce2 库实现硬件消抖，点击响应准确。

## 🛠️ 硬件需求

- 主控：ESP32（任何带有双核的型号，如 ESP32-WROOM-32）
- 运动传感器：MPU6050 (I2C)
- 按键：一个轻触开关（左键）
- 电池：单节锂电池（电压范围 1.8V ~ 2.5V 适配）
- 其他：10kΩ 电阻（测量电池电量，可选），USB-Micro 串口下载线

## 🔌 引脚连接

| 组件         | 引脚编号 |
|--------------|----------|
| 按键         | GPIO 4   |
| 电池电压检测 | GPIO 2   |
| MPU6050 SDA  | GPIO 21  |
| MPU6050 SCL  | GPIO 22  |

按键使用内部上拉，低电平有效。电池电压检测需将电池正极通过两个电阻（如 100kΩ 和 100kΩ）分压后接入 GPIO2，确保分压后电压不超过 3.3V。

## ⚙️ 软件安装

1. **安装 VSCode 与 PlatformIO 插件**  
   从 Visual Studio Code 扩展市场安装 PlatformIO IDE。

2. **克隆或创建项目**  
   新建一个 PlatformIO 项目，将提供的 `main.cpp` 放入 `src/` 目录。

3. **安装依赖库**  
   在 `platformio.ini` 中添加以下库依赖：
   ```ini
    lib_deps = 
        t-vk/ESP32 BLE Mouse @ ^0.3.1
        rfetick/MPU6050_light @ ^1.1.0
        thomasfredericks/Bounce2@^2.72
   ```
   并设置板型为 `esp32dev`，上传速度为 921600。

4. **编译与上传**  
   连接 ESP32，点击 PlatformIO 的 “Upload” 按钮。首次运行会自动校准陀螺仪，请保持设备水平静止。

## 🚀 快速开始

1. **上电**：ESP32 启动后，串口打印 “Init Done.”，同时蓝牙设备名 “AirMouse” 出现（可自行修改 `BleMouse` 构造函数中的名称）。
2. **配对**：在电脑/手机蓝牙设置中搜索 “AirMouse” 并连接。连接成功后蜂鸣器（如果有）或串口会输出 “[BLE] Connected”。
3. **使用**：
   - 移动设备：光标随之移动，角度灵敏度可通过 `GAIN` 等参数调节。
   - 按下按键：触发鼠标左键点击。
   - 电池电量：连接后自动上报一次，之后每 30 秒上报一次。
4. **断开**：关闭设备或主动断开蓝牙，任务会自动暂停并等待重连。

## 🔧 参数调校

所有核心参数均以 `#define` 定义在文件顶部，可根据硬件和手感需求修改。

### 运动控制参数

| 宏定义            | 默认值 | 说明                                                         |
| ----------------- | ------ | ------------------------------------------------------------ |
| `GAIN`            | 0.25   | 陀螺仪原始角速度（rad/s）到光标移动像素的转换增益。增大则移动更快。 |
| `DEAD_ZONE`       | 0.8    | 死区阈值（rad/s）。低于该值的微小抖动将被忽略。              |
| `SMOOTH_FACTOR`   | 0.5    | 指数平滑系数（0~1）。值越大对当前帧响应越灵敏，值越小越平滑。 |
| `INVERT_Y`        | 1      | Y 轴反转开关（1 反转，0 正常）。                             |

### 静止检测与零漂校准

| 宏定义                  | 默认值 | 说明                                                         |
| ----------------------- | ------ | ------------------------------------------------------------ |
| `STILL_WIN`             | 60     | 静止检测的滑动窗口长度（帧数）。                             |
| `RANGE_THRESH`          | 1.2    | 窗口内角速度极差阈值（rad/s）。小于该值视为候选静止。        |
| `ABS_THRESH`            | 2.0    | 窗口内角速度绝对均值阈值。                                   |
| `STILL_COUNT_THRESH`    | 80     | 连续满足静止条件的帧数，确认静止并开始更新偏置。             |
| `MOTION_LOCK_COUNT`     | 100    | 运动发生后锁定的帧数，期间不更新偏置。                       |
| `BIAS_ALPHA`            | 0.008  | 偏置更新的指数加权移动平均系数。                             |
| `ACCUM_CLEAR_MS`        | 1000   | 确认静止后等待多久清空未发送的累积位移（ms）。               |

### BLE 发送参数

| 宏定义                    | 默认值 | 说明                                                         |
| ------------------------- | ------ | ------------------------------------------------------------ |
| `PENDING_LIMIT`           | 2000   | 未发送位移的累积上限（像素）。超过后截断，防止蓝牙卡顿时产生巨大跳变。 |
| `BLE_SEND_INTERVAL_MS`    | 60     | BLE 发送任务的固定周期（ms）。60ms 对 Windows 最稳定；可改为 45ms 以提升 Mac/Android 流畅度。 |

## 📊 性能说明

- **采样率**：200Hz（每 5ms 读取一次 MPU6050）
- **延迟**：传感器采集到蓝牙发送的总延迟 = READ_INTERVAL(5ms) + 调度抖动(≈1ms) + BLE_SEND_INTERVAL(60ms) 的一半（平均 30ms） ≈ 36ms。实际手感流畅，无明显滞后。
- **功耗**：未连接时约 40mA，连接后约 60mA（取决于蓝牙发射功率和电池电压）。
- **精度**：静止时偏置可稳定在 ±0.5 rad/s 以内，移动时漂移率 < 1°/min。
